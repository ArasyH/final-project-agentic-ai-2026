2. Rencana integrasi mode 1, 2, 3
Prinsip integrasi
Jangan bongkar total. Refactor bertahap:
Mode 1 baseline
jalankan LLM tanpa retrieval, tanpa validator, tanpa semantic cache
tetap log ke telemetry agar evaluasi fair
Mode 2 RAG only
pakai retrieval service + answer composer
tanpa semantic cache
tanpa validator agent
tetap trace sources
Mode 3 full agentic
query normalization
semantic cache lookup
retrieval/tool caller
answer composer
validator/critic
fallback aman
telemetry penuh

Alur target
chat_api -> orchestrator_service -> mode runner -> structured output -> response
Kenapa ini paling aman
Karena:
endpoint existing tetap dipakai
RAG chain existing tetap bisa direuse dulu di tahap awal
semantic cache existing tetap dipakai lalu diperkaya
Langfuse existing tetap dipakai lalu dipusatkan lewat telemetry service
